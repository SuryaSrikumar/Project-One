# DRIVER™ Handbook
## AI-Augmented Professional Methodology

> **Cognition Mate (认知伙伴)** — Mutual help. Interdependent arising. Accomplishing together.

---

## 1. The Six Stages of DRIVER

| Stage | Purpose | Core Question | You're Done When |
|-------|---------|---------------|------------------|
| **D**efine | Scope & Research | What are we building and why? | Clear objective + known constraints |
| **R**epresent | Architectural Plan | How will we get there? | A plan someone else could follow |
| **I**mplement | Code & Execute | Let's build the solution | Working code that you can explain |
| **V**alidate | Evidence & Proof | Does it actually work? | Tested, reasonable, and defensible |
| **E**volve | Polish & Package | How can it be better? | Cleaner, documented, and reusable |
| **R**eflect | Capture Learnings | What did we learn? | Transferable insights captured |

---

## 2. Core Philosophy

1. **Pilot-in-Command**: AI is your co-pilot, not your autopilot. YOU hold the vision and verify outputs.
2. **Process Beats Everything**: A better process with a weaker AI/Human often beats a stronger AI/Human with no process.
3. **Logic is the Skill**: AI amplifies whatever you bring. Bring deep domain logic to avoid producing "confident-sounding garbage."
4. **Code as Bullshit Detector**: Code doesn't care about your confidence, only your correctness. It forces precision.

---

## 3. The "R-I" Loop (The Heart of DRIVER)
Real work is messier than a linear list. You will often:
- **REPRESENT**: Create a plan.
- **IMPLEMENT**: Start building.
- **LOOP BACK**: Building reveals a gap in the plan. Go back to REPRESENT, update the plan, then continue.

*This is not failure; it is the process working correctly.*

## 4. Claude Code Plugin Commands

If you have the DRIVER plugin installed:
- `/driver:init` — Start a new DRIVER project
- `/driver:status` — Check progress and get suggestions
- `/driver:define` — Begin research and definition stage
- `/driver:represent-roadmap` — Break your build into 3-5 sections
- `/driver:implement-screen` — Build and run specific code
- `/driver:validate` — Capture evidence and test results
- `/driver:evolve` — Package the final deliverable
- `/driver:reflect` — Record technical and domain learnings

## 5. Quick Reference: Effective AI Use

| Good (Bounded Tasks) | Dangerous (Vague/Open) |
|----------------------|------------------------|
| "Write a specific function for X" | "Build me a complete system for Y" |
| "Refactor this code for readability" | "Analyze this data and find something cool" |
| "Help me design test cases for Z" | "Write code for these loose requirements" |
